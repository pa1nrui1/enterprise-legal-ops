# 企业法务运营 Skill PRD

版本：v0.1.1  
卷号：第 1 卷  
状态：已确认基线  
文档主题：产品定义、用户引导、功能范围、总体验收标准

## 1. 产品定位

企业法务运营 Skill 是面向中小企业内部使用的本地法务运营管理 Skill。

它只服务企业内部管理，不生成律师顾问版，不生成正式法律意见书，不使用律师或律所身份对外出具文件。

它要解决的问题是：

1. 企业资料分散，合同、员工、证照、公章、章程等文件找不到。
2. 企业不知道哪些合同、劳动合同、证照、授权快到期。
3. 企业已有大量历史 Excel、PDF、Word 文件，缺少统一台账。
4. 老板、行政、HR 不知道某个事项下一步该怎么处理。
5. 企业需要在内部先做风险识别，再决定是否找外部专业人员或人工复核。

## 2. 首次安装引导

用户第一次安装或启用 Skill 时，必须出现清晰引导。

### 2.1 首次提示内容

Skill 应主动告诉用户：

```text
这个 Skill 可以帮你做四类事情：

1. 管合同：上传合同、生成合同台账、记录审查意见、提醒到期续签。
2. 管员工：导入员工花名册、劳动合同、考勤薪酬资料，生成 HR 法务台账。
3. 管公章证照：管理营业执照、许可证、资质、印章、授权文件、章程和合伙协议。
4. 管提醒：本地记录合同到期、劳动合同续签、证照年检、授权失效；如果配置飞书，可同步创建飞书日历提醒。

你可以从上传一个合同、上传员工花名册，或者建立公司基础档案开始。
```

### 2.2 首次配置项

首次配置至少包括：

1. 企业名称
2. 统一社会信用代码，可暂缺
3. 企业所在地
4. 主要经营地区
5. 是否使用飞书提醒
6. 本地资料保存根目录
7. 是否已有历史资料需要导入

### 2.3 首次配置标准

完成首次配置后，Skill 必须告诉用户：

1. 企业本地档案目录在哪里。
2. 公司基础信息记录在哪里。
3. 后续上传资料应如何归类。
4. 用户下一步可以做什么。

## 3. 目标用户

主要用户：

1. 企业老板
2. 行政负责人
3. HR
4. 财务
5. 综合管理人员

用户思维要求：

1. 当前正在处理什么。
2. 已经完成了什么。
3. 文件保存在哪里。
4. 台账在哪里查看。
5. 是否发现缺口或风险。
6. 下一步建议做什么。

每个任务结束时必须输出“处理完成说明”。

## 4. V1 功能范围

V1 包含五个模块：

1. 合同管理
2. 人力资源法务管理
3. 公章证照管理
4. 本地自然语言问库
5. 提醒模块

## 5. 提醒模块

提醒模块是独立模块，不影响其他模块运行。

无论是否使用飞书，都必须支持本地提醒台账。提醒事项包括：

1. 合同到期
2. 合同自动续约
3. 提前通知解除或不续约
4. 劳动合同到期
5. 试用期结束
6. 证照到期
7. 证照年检
8. 授权文件失效
9. 章程或合伙协议变更后需复核事项
10. 重要用印后续跟进事项

如果用户配置飞书，符合条件的提醒事项应同步创建飞书日历提醒。如果用户不使用飞书，不影响合同、HR、公章证照和本地问库，所有提醒只写入本地提醒台账。

## 6. 数据导入能力

V1 必须支持用户主动上传历史资料，不能假设所有数据都从 0 开始建立。

合同管理支持：

1. Word 合同
2. PDF 合同
3. 扫描件或图片合同
4. 历史合同台账 Excel
5. 补充协议、订单、附件

HR 管理支持：

1. 员工花名册 Excel
2. 劳动合同 Word/PDF
3. 员工手册
4. 薪酬制度
5. 考勤制度
6. 离职、处分、调岗、降薪材料

公章证照管理支持：

1. 营业执照
2. 许可证
3. 资质证书
4. 公司章程
5. 合伙协议
6. 股东会决议
7. 董事会决议
8. 授权委托书
9. 印章清单
10. 用印登记表

上传资料后，Skill 必须读取文件内容、提取关键字段、生成或更新 CSV 台账、生成 Markdown 详情记录、保存原始文件路径、标注读取方式和存疑项，并告诉用户台账和记录文件在哪里。

## 7. 企业内部身份限制

本 Skill 永远只输出企业内部工作文件。

禁止：

1. 输出律师顾问版。
2. 输出正式法律意见书。
3. 使用律师姓名或律所名称落款。
4. 将内部风险判断包装为律师正式意见。
5. 将未经人工复核的结论对外使用。

允许输出：

1. 企业内部风险提示
2. 企业内部合同审查记录
3. 企业内部制度修改建议
4. 企业内部用印风险提示
5. 企业内部提醒台账
6. 企业内部资料归档记录

## 8. 合同管理用户流程

用户上传合同后，Skill 必须识别合同文件、确认合同归属、确认合同审查立场、保存原始合同、生成合同编号、写入合同台账、生成合同概览、提取到期/续约/提前通知事项、写入提醒台账，并告诉用户本次处理结果和下一步。

每次合同处理完成后，必须说明：

1. 合同原件保存位置
2. 合同台账位置
3. 合同概览记录位置
4. 提醒台账位置
5. 本次发现的关键提醒
6. 下一步建议

## 9. HR 管理用户流程

用户上传员工花名册后，Skill 必须读取表格、识别员工姓名/部门/岗位/入职时间/合同起止日期/试用期等字段、生成员工台账、为员工生成或更新 Markdown 记录、提取试用期结束/劳动合同到期/续签提醒、写入提醒台账，并告诉用户缺失字段。

用户上传历史劳动合同后，Skill 必须读取劳动合同内容、匹配员工花名册、无法匹配时提示用户确认、提取合同期限/岗位/工资/工作地点/试用期/续签条款、更新员工记录和提醒事项。

## 10. 公章证照管理用户流程

公章证照模块不仅管理证照和印章，还必须管理企业基础治理文件，包括：

1. 公司章程
2. 合伙协议
3. 股东会决议
4. 董事会决议
5. 法定代表人授权文件
6. 印章管理制度
7. 对外担保制度
8. 财务审批制度

上传章程或合伙协议后，Skill 必须提取公司名称、股东或合伙人、法定代表人或执行事务合伙人、对外投资权限、对外担保权限、借款或融资权限、重大合同审批权限、印章使用审批权限、表决比例、版本日期、生效日期和后续修正案。

每次章程或合伙协议变更，必须生成变更记录，并提示是否影响担保、借款、重大合同、用印审批、历史授权或审批事项。

涉及对外承诺、担保、借款、融资、股权转让、重大资产处置、长期或大额合同、劳动解除或群体性员工事项、授权他人代表公司签署文件时，必须与章程、合伙协议或内部审批制度对比。

## 11. 自然语言问库

自然语言问库必须支持从上传后的历史资料中查询。示例：

1. 下个月有哪些合同到期？
2. 哪些员工劳动合同快到期？
3. 公司章程里对外担保是谁审批？
4. 某某合同原件在哪里？
5. 某个员工的劳动合同有没有上传？
6. 哪些证照今年需要年检？
7. 最近一次章程变更改了什么？
8. 某份担保合同用印前需要什么审批？

回答必须来自本地台账或文件，必须标注来源，没查到时必须说明未查到，涉及法律判断时必须进入严审。

## 12. 总体验收标准

V1 完成时必须满足：

1. 首次安装时能完成企业基础档案初始化。
2. 能清楚告诉用户 Skill 能解决什么问题。
3. 能支持用户上传既有 Excel、PDF、Word、图片资料。
4. 能从上传资料中生成 CSV 台账和 Markdown 记录。
5. 能告诉用户文件保存位置、台账位置和下一步建议。
6. 能管理合同、员工、公章、证照、章程、合伙协议。
7. 能记录章程和合伙协议变更。
8. 能在担保、借款、承诺、重大合同等事项中对比章程权限。
9. 能维护本地提醒台账。
10. 如果配置飞书，能同步创建飞书日历提醒。
11. 如果不配置飞书，其他模块不受影响。
12. 所有输出均为企业内部文件，不进入律师顾问版或正式法律意见版本。

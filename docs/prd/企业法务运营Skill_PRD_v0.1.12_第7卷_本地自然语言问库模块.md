# 企业法务运营 Skill PRD

版本：v0.1.12  
卷号：第 7 卷  
状态：已确认基线  
文档主题：本地自然语言问库模块 PRD

## 1. 本卷目标

本地自然语言问库模块用于让企业用户用普通语言查询本地合同、员工、制度、证照、公章、授权、章程、股东出资和提醒信息。

它要解决企业资料很多但不知道文件放在哪里、台账字段很多但用户不想手动筛选 CSV、用户希望直接问本地资料、系统必须区分管理型查询和法律判断型问题等问题。

## 2. 模块边界

本模块包含自然语言查询解析、本地 CSV 台账查询、Markdown 详情记录查询、读取文本全文检索、原始文件路径定位、历史处理记录查询、提醒事项查询、来源标注、查不到时的缺口提示、法律判断型问题转入对应模块严审。

本模块不包含联网搜索、法律法规核验、实质合同审查、HR 风险预判、章程权限判断、飞书同步、生成正式文件。如果问题涉及法律判断，本模块只负责定位资料并路由。

## 3. 目录结构

```text
<company_workspace>/05-本地问库/
├── search-index.md
├── qa-log.md
├── source-map.csv
├── extracted-text-index.csv
├── query-routing-log.md
└── unresolved-queries.md
```

## 4. 查询类型

管理型查询可以直接通过本地台账或记录回答，例如某份合同在哪里、哪些合同下个月到期、哪些员工试用期快结束、某员工劳动合同有没有上传、公司有哪些证照今年要年检、某枚公章由谁保管、某股东出资什么时候到期、哪些提醒还没处理。

历史记录查询需要查询过去生成的记录，例如某合同之前审查过哪些问题、某员工以前有没有解除风险记录、上次章程变更改了什么、某制度上次合法性审查发现什么问题、某用印事项有没有做过权限校验。

法律判断型问题不能直接由问库回答，必须路由到对应模块严审，例如这份合同能不能签、这个员工能不能辞退、这个制度合法吗、这个担保能不能盖章、未休年假要不要补偿、这个授权是否有效。

## 5. 查询路由规则

合同相关优先查询 `contracts.csv`、`contract-record.md`、合同读取文本、审查记录、版本记录、提醒台账。

HR 相关优先查询 `employees.csv`、`employment-contracts.csv`、`annual-leave.csv`、`employee-record.md`、制度索引、制度审查记录、提醒台账。

公章证照相关优先查询 `licenses.csv`、`seals.csv`、`authorizations.csv`、`seal-use.csv`、`governance-documents.csv`、`capital-contributions.csv`、`authority-checks.csv`、章程权限摘要、提醒台账。

提醒相关优先查询 `reminders.csv`、`reminder-summary.md`、`feishu-sync-log.md`。

## 6. 查询执行顺序

```text
理解用户问题
→ 判断查询类型
→ 判断所属模块
→ 查询对应 CSV 台账
→ 查询 Markdown 详情记录
→ 查询读取文本
→ 必要时定位原始文件
→ 判断是否需要严审
→ 生成回答
→ 标注来源
→ 记录问答日志
```

能用 CSV 回答的，不全文扫所有文件；CSV 信息不足时再查 Markdown；Markdown 信息不足时再查读取文本；查不到时不得编造；涉及法律判断时不得只靠台账回答。

## 7. 来源映射

`source-map.csv` 字段：

```text
source_id,module,object_id,object_name,csv_path,md_path,extracted_text_path,original_file_path,last_updated_at
```

每个合同、员工、证照、授权、用印、章程、出资事项都应有来源映射。

## 8. 读取文本索引

`extracted-text-index.csv` 字段：

```text
text_id,module,object_id,source_file_path,extracted_text_path,read_method,read_status,ocr_status,key_fields,created_at,updated_at
```

用于快速定位读取文本、支持全文检索、标注 OCR 或读取状态、避免重复读取同一文件。

## 9. 回答格式

管理型查询回答格式：

```markdown
## 查询结果

- 查询问题：
- 查询范围：
- 找到结果：
- 来源：
- 下一步建议：
```

历史记录查询回答格式：

```markdown
## 历史记录查询结果

- 查询问题：
- 已找到记录：
- 关键摘要：
- 来源文件：

```

法律判断型问题回答格式：

```markdown
## 需要进入严审

这个问题涉及实质法律判断，不能仅根据台账直接回答。

已找到资料：
-

还需要补充或确认：
-

建议进入模块：
-

下一步：
-
```

## 10. 查不到时的处理

没有查到结果时，必须说明查了哪些台账、查了哪些记录、没查到什么、可能原因、用户下一步可以上传或确认什么，并写入：

```text
<company_workspace>/05-本地问库/unresolved-queries.md
```

## 11. 问答日志

所有自然语言问答应记录到：

```text
<company_workspace>/05-本地问库/qa-log.md
```

记录查询时间、用户问题、查询类型、查询范围、命中的文件、是否进入严审、回答摘要、未解决事项。

## 12. 严审路由

合同严审包括能不能签、有没有风险、怎么修改、是否自动续约有效、是否构成违约、是否能解除、是否需要盖章或授权。

HR 严审包括能不能解除、能不能调岗、能不能降薪、能不能处分、加班费怎么算、未休年假是否补偿、制度是否合法、新旧制度是否冲突。

公章证照严审包括能不能盖章、授权是否有效、担保是否需要股东会、借款是否需要董事会、出资是否逾期影响交易、证照是否覆盖业务资质。

## 13. 用户完成提示

每次问库查询结束必须输出查询范围、找到的结果、来源文件、是否需要进入严审、下一步建议。不得只回复查询结果，不说明来源和下一步。

## 14. 验收标准

1. 能识别管理型查询、历史记录查询、法律判断型问题。
2. 能查询合同、HR、公章证照、股东出资、提醒等 CSV 台账。
3. 能查询对应 Markdown 详情记录。
4. 能定位原始文件路径。
5. 能引用读取文本回答原文类问题。
6. 能标注来源。
7. 查不到时不编造，并记录未解决问题。
8. 法律判断型问题能转入对应模块严审。
9. 能记录问答日志。
10. 每次查询完成后告诉用户查询范围、来源文件、是否需要严审和下一步。

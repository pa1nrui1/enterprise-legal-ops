# 企业法务运营 Skill PRD

版本：v0.1.13  
卷号：第 8 卷  
状态：已确认基线  
文档主题：严审流程与用户引导规范

## 1. 本卷目标

本卷定义企业法务运营 Skill 中所有模块共同遵守的严审流程和用户引导规范。

它要解决系统不能把未经读取、未经核验的信息包装成确定结论，用户必须知道系统正在处理什么、已完成什么、文件保存在哪里，涉及法律判断的事项必须进入严审，系统输出必须清楚区分事实、风险、建议、缺口和下一步，企业内部使用场景下不能生成律师顾问版或正式法律意见书。

## 2. 全局身份限制

企业法务运营 Skill 永远只输出企业内部工作文件。

禁止输出律师顾问版、正式法律意见书、律师姓名或律所名称落款、将内部风险判断包装为律师正式意见、将未经人工复核的结论作为对外文件、生成法院/仲裁委/行政机关提交材料。

允许输出企业内部风险提示、企业内部审查记录、企业内部修改建议、企业内部 Word 修订稿、企业内部制度审查记录、企业内部用印权限校验记录、企业内部提醒台账、企业内部资料归档记录。

重大事项可以提示：

```text
该事项风险较高，建议企业在对外签署、解除、担保、借款、出资、用印或作出重大承诺前，由负责人或外部专业人员复核。
```

## 3. 严审触发范围

合同类严审包括合同审查、修改建议、Word 修订稿、合同起草、合同解除、合同续签、补充协议、担保借款融资股权重大资产长期合同、是否能签署、是否能解除、是否构成违约。

HR 类严审包括员工解除、调岗、降薪、处分、离职争议、加班费计算、未休年假补偿、工资扣款、工时制度、旧制度合法性审查、新旧制度冲突检查、新制度起草或修改、规章制度生效程序判断。

公章证照类严审包括能否盖章、授权是否有效、担保审批权限、借款审批权限、融资审批权限、重大合同审批权限、章程权限判断、章程变更影响判断、股东出资逾期风险判断、资质是否覆盖拟开展业务。

问库类管理型查询不需要严审；涉及法律判断、风险判断、金额计算、权限判断、签署建议、解除建议、制度合法性判断时，必须转入对应模块严审。

## 4. 严审总流程

```text
识别任务类型
→ 确认用户目标
→ 确认关键立场或适用场景
→ 完整读取材料
→ 提取关键事实
→ 读取复查
→ 法规或规则核验
→ 风险识别
→ 生成建议
→ 输出来源边界
→ 写入本地记录
→ 告诉用户文件位置和下一步
```

不得跳过材料读取和来源边界。

## 5. 用户目标确认

合同任务必须确认审查还是起草、我方立场、是否需要 Word 修订稿、是否涉及用印授权担保借款融资重大承诺、是否存在签署截止时间。

HR 任务必须确认是管理查询、制度审查还是具体员工风险处理，涉及哪个员工或哪份制度，员工工作地或主要履行地，企业拟采取的动作，是否已有事实材料、制度文件和程序材料。

公章证照任务必须确认是证照管理、用印、授权还是权限判断，涉及哪份文件，是否涉及担保借款融资重大合同出资对外承诺，当前有效章程或合伙协议是否已上传，是否已有决议或授权材料。

## 6. 完整读取材料

用户上传的所有相关文件必须逐一读取。支持 `.docx`、`.doc`、`.pdf`、`.xlsx`、`.xls`、`.csv`、`.jpg`、`.jpeg`、`.png`、`.txt`、`.md`。

每个文件必须记录文件名、文件路径、文件类型、读取方式、读取是否成功、提取的关键字段、存疑项、是否需要用户确认。

读取失败时必须告诉用户哪个文件失败、尝试了什么方法、失败原因、是否需要 OCR、是否需要重新上传、该失败会影响什么结论。

## 7. 读取复查

读取完成后，必须复查主体名称、日期、金额、身份证号或统一社会信用代码、合同编号、员工编号、证照编号、出资金额、出资期限、授权范围、审批权限、OCR 存疑字段。

读取复查摘要格式：

```markdown
# 读取复查摘要

## 文件
-

## 读取方式
-

## 关键字段
-

## 存疑项
-

## 完整性评估
-

## 是否需要用户确认
-
```

## 8. 法规或规则核验

凡涉及法律法规、地方规则、监管要求、强制性规定、资质要求、劳动规则、公司治理规则时，必须核验。

必须核验的场景包括合同效力判断、担保借款融资股权出资规则、劳动解除调岗降薪处分、工资工时加班费未休年假、规章制度合法性、证照资质适用范围、章程权限与公司法规则、授权有效性判断。

法规或规则核验摘要格式：

```markdown
# 法规/规则核验摘要

## 核验事项
-

## 核验来源
-

## 核验结果
-

## 适用地区
-

## 现行有效状态
-

## 对本事项的影响
-
```

不得用模型记忆替代核验。

## 9. 来源边界

所有严审输出必须包含来源边界：

```markdown
# 来源边界

## 已读取材料
-

## 已核验内容
-

## 未读取或缺失材料
-

## 存疑项
-

## 输出边界
-
```

输出边界必须说明哪些结论来自文件、用户口述、法规或规则核验，哪些事项需要用户确认，哪些事项不能作为对外正式意见使用。

## 10. 风险等级

全局风险等级使用：

```text
低
中
高
暂停处理
```

合同模块可使用：

```text
低
中
高
签署阻断
```

权限校验模块使用：

```text
已通过
待补充审批
章程无明确依据
建议暂停用印
待人工确认
```

## 11. 输出格式总规范

严审输出必须包含事项摘要、已读取材料、关键事实、风险判断、修改或处理建议、缺失材料、来源边界、下一步建议、文件保存位置。不得只输出结论。

## 12. 用户引导规范

开始处理时必须告诉用户：

```text
我将先读取你提供的文件，提取关键字段，并检查是否需要补充材料。处理完成后，我会告诉你文件保存位置、台账位置、发现的风险和下一步建议。
```

处理中发现缺口时必须告诉用户：

```text
当前缺少以下信息，这会影响结论准确性：
...

我可以先按现有材料生成内部风险提示，但会把这些内容标记为待确认。
```

处理完成时必须输出已完成事项、生成或更新的文件、台账位置、发现的风险或缺口、下一步建议。

## 13. 本地记录要求

严审过程必须写入本地记录，至少包括读取复查摘要、法规/规则核验摘要、来源边界记录、用户确认记录、风险判断记录、输出文件路径。

## 14. 用户确认记录

合同审查立场、合同最终版确认、是否使用模板起草、员工处理动作、工作地或适用地区、是否存在特殊保护状态、是否已有民主程序/公示/签收、用印是否继续、是否已有审批或授权、是否启用飞书提醒、是否接受信息缺失情况下的内部初步提示必须记录用户确认。

用户确认记录格式：

```markdown
# 用户确认记录

## 确认时间
-

## 确认事项
-

## 用户选择
-

## 对输出的影响
-
```

## 15. 禁止事项

系统不得在未读取材料时输出确定结论、在未核验法规时引用具体法律规则、在未确认审查立场时审查合同、在未读取制度时判断解除处分扣款合法、在未读取章程或授权时判断可以担保借款盖章、将内部文件包装为正式法律意见、使用律师或律所身份、生成法院仲裁委行政机关提交材料、因飞书同步失败而丢失本地提醒、覆盖原始文件、删除历史版本或历史提醒、编造未查到的资料。

## 16. 验收标准

1. 所有模块能识别是否需要严审。
2. 严审前能确认用户目标和关键立场。
3. 能完整读取用户材料。
4. 能生成读取复查摘要。
5. 能进行法规或规则核验。
6. 能生成来源边界。
7. 能记录用户确认事项。
8. 能输出风险等级和处理建议。
9. 能告诉用户文件位置、台账位置和下一步。
10. 不生成律师顾问版或正式法律意见。
11. 不覆盖原始文件。
12. 不把未读取、未核验、未确认的信息当作确定事实。

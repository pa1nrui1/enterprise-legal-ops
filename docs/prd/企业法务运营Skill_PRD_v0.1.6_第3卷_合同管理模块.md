# 企业法务运营 Skill PRD

版本：v0.1.6  
卷号：第 3 卷  
状态：已确认基线  
文档主题：合同管理模块 PRD

## 1. 本卷目标

合同管理模块用于帮助企业完成合同的上传、归档、台账生成、风险审查、Word 修订稿生成、合同起草、模板库管理、版本管理、到期提醒和历史问答。

它要解决的问题是：

1. 合同原件分散，无法快速找到。
2. 合同没有统一编号和台账。
3. 合同到期、自动续约、提前通知容易遗漏。
4. 合同审查意见没有沉淀。
5. 多个合同版本关系不清。
6. 企业不知道合同签署前还需要补什么材料或审批。
7. 企业希望审查后直接拿到可修改、可批注、可流转的 Word 修订稿。

## 2. 模块边界

合同管理模块包含：

1. 合同上传与归档
2. 合同台账
3. 合同详情记录
4. 合同审查
5. Word 修订格式审核稿
6. 合同起草
7. 企业自有模板库
8. 合同版本管理
9. 合同最终版确认
10. 合同提醒
11. 合同自然语言问答
12. 章程权限联动

合同管理模块不包含：

1. 飞书合同审查同步
2. 业务流程图
3. 合同思维导图
4. 简易谈判底线
5. 律师顾问意见
6. 正式法律意见书
7. 律师或律所落款文件

飞书只在“提醒模块”中使用，不参与合同审查交付。

## 3. 用户入口

用户可以通过以下方式触发合同管理模块：

1. 上传一个或多个合同文件。
2. 上传历史合同台账。
3. 上传企业历史模板或定稿合同。
4. 询问某份合同在哪里。
5. 询问合同是否快到期。
6. 要求审查合同风险。
7. 要求生成 Word 修订稿。
8. 要求起草合同。
9. 上传合同新版本。
10. 表示某份合同是最终版、定稿版或签署版。
11. 询问某份合同以前审查过什么问题。

## 4. 合同目录结构

合同管理目录位于：

```text
<company_workspace>/01-合同管理/
```

建议结构：

```text
01-合同管理/
├── contracts.csv
├── contract-reminders.csv
├── 模板库/
│   ├── template-index.csv
│   └── <template_id-模板名称>/
├── <合同编号-合同简称>/
│   ├── 01-原始文件/
│   ├── 02-读取文本/
│   ├── 03-合同记录/
│   ├── 04-审查记录/
│   ├── 05-版本记录/
│   └── 06-输出文件/
```

## 5. 合同上传流程

用户上传合同后，系统必须按顺序完成：

1. 保存原始文件到导入暂存区。
2. 识别文件格式。
3. 完整读取合同内容。
4. 提取合同基础字段。
5. 判断是否已有相同或相似合同记录。
6. 询问或确认合同归属项目。
7. 生成合同编号。
8. 保存原始文件到合同目录。
9. 生成或更新合同台账。
10. 生成合同 Markdown 详情记录。
11. 提取期限、续约、提前通知条款。
12. 写入本地提醒台账。
13. 如飞书提醒已配置，将提醒事项交给提醒模块处理。
14. 输出处理完成说明。

## 6. 合同基础字段提取

系统应尽量从合同中提取合同名称、合同类型、我方主体、我方角色、相对方主体、合同金额、签署日期、生效日期、履行期限、自动续约、提前通知期限、付款节点、交付或服务内容、验收条款、违约责任、解除条件、争议解决、附件订单补充协议情况、是否涉及用印、是否涉及担保借款融资重大承诺。

字段提取标准：

1. 能从合同明确读取的，填写原文事实。
2. 不能读取的，标记 `待确认`。
3. OCR 存疑的，标记 `OCR待复核`。
4. 不得靠模型猜测金额、期限、主体或续约条款。
5. 关键字段必须记录来源条款位置。

## 7. 合同编号规则

每份合同必须生成唯一合同编号。推荐格式：

```text
合同类型-相对方简称-序号
```

合同编号一经生成，不得因合同改名而改变；编号冲突时必须提示用户确认。

## 8. 合同台账

合同台账文件：

```text
<company_workspace>/01-合同管理/contracts.csv
```

合同台账字段至少包括：

```text
contract_id,contract_name,contract_type,our_party,our_role,counterparty,review_position,amount,effective_date,end_date,auto_renewal,notice_days,latest_notice_date,status,original_file_path,record_md_path,latest_review_date,risk_level,next_action,created_at,updated_at
```

## 9. 合同详情记录

每份合同必须生成：

```text
<合同编号-合同简称>/03-合同记录/contract-record.md
```

合同详情记录必须包括基础信息、文件位置、关键条款摘要、期限和提醒、审查记录索引、版本记录索引、来源边界、下一步动作。

## 10. 合同审查触发

以下情况进入合同审查：

1. 用户明确说“审查合同”。
2. 用户问“这份合同有没有风险”。
3. 用户要求“帮我改合同”。
4. 用户要求“生成修改建议”。
5. 用户要求“能不能签”。
6. 合同涉及高风险事项。
7. 用户要求生成 Word 修订稿。

## 11. 合同审查前置确认

审查前必须确认审查立场、企业在合同中的角色、关注重点、是否需要 Word 修订稿、是否涉及担保借款融资重大合同或对外承诺、是否需要联动章程权限校验、是否存在签署截止时间。

未确认审查立场前，不得输出实质审查意见。

## 12. 合同审查方法

合同审查参考既有 `合同审查 Skill` 的审查方法，但按企业内部使用场景简化交付。

每份合同审查至少包括：

1. 交易结构审查
2. 合同类型识别
3. 条款完整性审查
4. 专项风险审查

系统必须识别谁付款、谁交付、交付什么、如何验收、风险何时转移、违约后如何救济，以及是否涉及用印、授权、担保、借款、融资、重大承诺。

至少支持公司及股权、买卖采购、租赁、承揽劳务服务、中介推广居间、知识产权、劳动用工、担保、借款、特许经营加盟、合作、房地产建设工程、软件 IT SaaS、物流运输、保密协议/NDA、补充解除和解债务确认、续约自动续约到期提醒类合同。

每份合同至少检查主体资格、签约权限、合同标的、价款付款、履行期限、交付验收、违约责任、解除终止、争议解决、通知送达、附件补充文件、用印授权要求、到期续约提前通知。

## 13. 风险等级

合同风险等级分为：

```text
低
中
高
签署阻断
```

签署阻断事项包括主体不存在或明显不适格、签约权限或用印权限缺失、对外担保/借款/融资/重大承诺未取得章程要求审批、核心交易条款缺失、付款交付验收解除违约责任明显失衡、涉嫌违法或明显不可执行、企业经营资质不足以履行合同。

## 14. 审查问题格式

```markdown
### 问题 001｜问题标题

- 条款位置：
- 原文摘录：
- 问题类型：
- 风险说明：
- 处理建议：必须修改 / 建议修改 / 需企业确认 / 可优化
- 建议修改文本：
- 是否进入 Word 修订稿：
- 来源：
```

企业版不输出对方不同意时、可接受底线、谈判让步空间、律师策略分析。

## 15. 合同审查唯一交付物

合同审查完成后，默认且唯一交付：

```text
Word 修订格式合同审核稿
```

不得输出飞书合同审查文档、业务流程图、合同思维导图、简易谈判底线、律师顾问意见、正式法律意见书、律师或律所落款文件。

## 16. Word 修订稿要求

Word 修订稿必须满足：

1. 原合同只作为只读来源。
2. 不覆盖原合同。
3. 另存为新的审核修订稿。
4. 使用 Word 真实修订痕迹。
5. 使用批注说明重大风险、事实缺口或需企业确认事项。
6. 对确定性文字问题、编号问题、主体名称问题、交叉引用问题可直接修订。
7. 对金额、期限、付款节奏、责任上限等商务取舍，不得替企业拍板，只能批注提示确认。
8. 修订完成后必须进行结构检查。
9. 输出完成后必须告诉用户文件保存位置。

## 17. Word 修订稿技术路径

流程：

```text
完整读取合同
→ 提取合同基础信息
→ 识别合同类型
→ 识别通用风险和专项风险
→ 形成审查问题清单
→ 生成 redline-plan
→ 将问题分配为 replace / insert / delete / comment / report-only
→ 生成 Word 真实修订痕迹
→ 生成批注
→ QA 检查修订结构
→ 检查接受修订后的清洁文本
→ 输出企业内部 Word 修订稿
```

必须使用真实 Word 修订结构，不得用颜色、删除线、下划线伪造；未命中、重复命中、定位不确定的修改不得强行落痕；留空项、金额、日期、联系人、账号等不得由模型猜测补写。

## 18. 禁止生成图形化交付物

合同审查模块不生成业务流程图、合同思维导图、Mermaid 图、飞书流程图或任何图形化合同审查交付物。

如果用户要求理解合同流程，只能用文字摘要说明，不生成图。

## 19. 合同起草

合同起草参考既有 `合同起草 Skill`。支持企业自有模板起草、通用模板起草、独立起草，优先顺序为：

```text
企业自有模板
→ 通用模板
→ 独立起草
```

起草前必须确认合同类型、我方主体、相对方、我方立场、交易背景、标的或服务内容、金额和付款方式、履行期限、交付或验收方式、违约责任偏好、争议解决方式、是否需要用印、是否涉及担保借款授权重大承诺、是否使用企业自有模板。

合同草案必须使用企业内部文件身份，不出现律师或律所落款，条款结构完整，编号清晰，缺失信息标记 `待确认`，明确提示需企业确认的商务事项。

## 20. 企业自有模板库

企业可以上传历史合同、常用模板、曾经定稿的合同，作为企业自有模板库。保存位置：

```text
<company_workspace>/01-合同管理/模板库/
```

模板索引文件：

```text
<company_workspace>/01-合同管理/模板库/template-index.csv
```

字段包括：

```text
template_id,template_name,contract_type,preferred_role,applicable_scene,source_file_path,record_md_path,last_used_at,status,created_at,updated_at
```

## 21. 合同版本管理

合同版本包括初始上传版、AI 起草版、审查修改版、对方反馈版、谈判版、定稿版、签署版、终止版、补充协议、续签协议。

每个版本必须记录版本号、版本来源、文件路径、形成时间、基于哪个版本、本次变化摘要、是否改变金额/期限/付款/违约/解除/争议解决、是否影响提醒、是否为最终版。

只有用户明确说最终版、定稿、签署版、已签署、归档时，才能标记最终版；不得根据文件名自行判断最终版。

## 22. 合同提醒

V1 必须至少支持合同到期、自动续约、提前通知。其他节点如果能准确提取，也应写入本地提醒台账。飞书提醒由提醒模块处理，不属于合同审查交付。

## 23. 合同审查与章程权限联动

担保合同、借款合同、融资合同、股权转让合同、重大资产处置合同、长期或大额合同、授权代表公司签署的文件、对外承诺函、涉及公司治理事项的协议必须联动公章证照模块。

联动检查章程或合伙协议是否有权限规定，是否需要股东会/董事会/执行董事/合伙人会议批准，是否已有决议或授权文件，用印是否符合内部制度，是否建议暂停签署或用印。

## 24. 合同自然语言问答

用户可以询问某份合同在哪里、什么时候到期、是否自动续约、以前审查过哪些问题、某个相对方有多少合同、某类合同总金额、最终版是哪一个、有没有生成过 Word 修订稿。

问答标准：先查合同台账，再查合同详情记录，必要时查读取文本；回答必须标注来源；查不到不得编造；涉及法律判断时进入合同审查流程。

## 25. 合同任务完成提示

每次合同任务结束必须输出：

```text
本次合同处理已完成。

已完成事项：
...

合同原件位置：
...

合同台账位置：
...

合同详情记录：
...

...

本次新增提醒：
...

发现的风险或缺口：
...

你下一步可以：
1. ...
2. ...
3. ...
```

## 26. 验收标准

1. 能上传并读取 Word、PDF、图片合同。
2. 能生成唯一合同编号。
3. 能写入合同台账。
4. 能生成合同详情 Markdown。
5. 能提取合同期限、续约和提前通知提醒。
6. 能识别合同类型并加载对应风险审查规则。
7. 能参考既有合同审查 Skill 的方法完成风险识别。
8. 能在审查前强制确认审查立场。
9. 合同审查唯一交付物为 Word 修订格式审核稿。
10. Word 修订稿必须是真实修订痕迹，不得伪造红线。
11. 合同审查不得同步到飞书。
12. 合同审查不得生成业务流程图或思维导图。
13. 合同审查不得输出谈判底线、律师顾问版或正式法律意见。
14. 能支持企业自有合同模板库。
15. 合同起草优先使用企业自有模板。
16. 能管理合同多个版本。
17. 能确认最终版但不自行猜测最终版。
18. 涉及担保、借款、融资、重大承诺时，能联动章程权限校验。
19. 能回答合同相关自然语言问题。
20. 每次处理完成后能告诉用户原合同位置、修订稿位置、台账位置、提醒状态和下一步。

# 企业法务运营 Skill PRD

版本：v0.1.10  
卷号：第 5 卷  
状态：已确认基线  
文档主题：公章证照与章程权限模块 PRD

## 1. 本卷目标

公章证照与章程权限模块用于帮助企业管理营业执照、许可证、资质证书、印章、授权文件、章程、合伙协议、股东会决议、董事会决议、股东出资等基础治理文件，并在用印、签约、担保、借款、重大承诺等事项中进行权限校验。

它要解决的问题：

1. 企业证照分散，过期、年检、变更事项容易遗漏。
2. 企业印章管理没有清晰台账。
3. 用印记录不完整，事后难以追溯。
4. 授权文件范围不清，过期或撤销后仍被使用。
5. 章程、合伙协议、决议等治理文件没有结构化记录。
6. 对外担保、借款、重大合同、承诺函等事项不知道是否需要审批。
7. 章程变更后，没有同步检查旧授权、旧审批和历史用印风险。
8. 股东认缴出资期限临近或逾期无人提醒。

## 2. 模块边界

本模块包含证照管理、资质管理、印章管理、用印记录、授权文件管理、公司章程管理、合伙协议管理、股东会/董事会/执行董事决定等决议管理、章程权限摘要、章程变更记录、用印和签约权限校验、股东出资管理、高风险事项提醒、公章证照自然语言问答。

本模块不包含工商变更代理、资质许可申报代理、对外法律意见、律师或律所落款文件、诉讼或仲裁材料。

## 4. 用户入口

用户可以上传营业执照、许可证或资质证书、公司章程、合伙协议、章程修正案、股东会/董事会/执行董事决定、授权委托书、印章清单、用印登记表、出资证明、验资报告、银行回单；也可以询问证照到期、授权有效性、章程中担保/借款/重大合同如何审批、某份文件能否盖章、某次用印是否需要补充审批、股东出资是否到期或逾期。

## 5. 证照管理

证照包括营业执照、行业许可证、资质证书、备案证明、食品/医疗/教育/建筑/运输等行业证照、法定代表人身份证明、开户许可证或银行基本户信息、其他企业经营所需证照。

用户上传证照后，系统必须保存原始文件、完整读取证照内容、提取证照名称/编号/主体/发证机关/有效期等字段、判断证照主体是否与企业主体一致、写入 `licenses.csv`、生成证照 Markdown 记录、提取到期/年检/换证提醒、写入提醒台账。

`licenses.csv` 字段至少包括：

```text
license_id,license_name,license_type,holder,license_number,issue_authority,issue_date,expiry_date,annual_check_required,annual_check_deadline,file_path,record_md_path,status,risk_flags,next_action,created_at,updated_at
```

证照已过期、即将到期、主体不一致、经营范围不匹配、年检时间缺失、无法确认真伪或有效状态、OCR 日期编号存疑时必须提示风险。

## 6. 印章管理

至少支持公章、合同专用章、财务专用章、发票专用章、法定代表人名章、部门章、项目章、电子印章。

`seals.csv` 字段至少包括：

```text
seal_id,seal_name,seal_type,custodian,authorized_scope,approval_rule,storage_location,status,record_md_path,created_at,updated_at
```

每枚印章必须有唯一编号、保管人、用途范围、保管地点、适用审批规则。印章停用、遗失、换刻不得删除原记录，只能变更状态。

## 7. 用印管理

`seal-use.csv` 字段至少包括：

```text
seal_use_id,seal_id,document_name,document_type,purpose,applicant,approver,use_date,counterparty,amount,high_risk_type,authority_check_status,governance_basis,file_path,record_md_path,next_action,created_at,updated_at
```

高风险用印包括对外承诺、担保、借款、融资、股权转让、重大资产处置、长期合同、大额合同、劳动解除、授权他人代表公司签署文件、其他可能对企业产生重大义务的文件。

高风险用印必须读取拟盖章文件、识别文件性质、提取金额/相对方/义务内容、联动章程或合伙协议进行权限校验、检查是否已有决议或授权、生成权限校验记录；权限依据不足时提示暂停用印或补充审批。

## 8. 授权文件管理

授权文件包括授权委托书、法定代表人授权、业务授权、签约授权、用印授权、对外谈判授权、项目负责人授权、电子签章授权。

`authorizations.csv` 字段至少包括：

```text
authorization_id,authorization_name,grantor,grantee,scope,start_date,end_date,revocation_status,file_path,record_md_path,risk_flags,created_at,updated_at
```

必须记录授权主体、被授权人、授权范围、授权期限；有期限的授权必须进入提醒；授权范围不明确时必须标记风险；撤销或失效后不得删除原记录，只能更新状态。

## 9. 章程与合伙协议管理

必须管理公司章程、章程修正案、合伙协议、合伙协议补充协议、股东会决议、董事会决议、执行董事决定、合伙人会议决议、对外担保制度、财务审批制度、印章管理制度、重大合同审批制度。

`governance-documents.csv` 字段至少包括：

```text
governance_doc_id,doc_name,doc_type,version_date,effective_date,company_name,key_authority_summary,file_path,record_md_path,status,created_at,updated_at
```

上传章程或合伙协议后，系统必须提取公司或企业名称、股东或合伙人、法定代表人或执行事务合伙人、股东会权限、董事会权限、执行董事或经理权限、合伙人会议权限、对外投资权限、对外担保权限、借款或融资权限、重大合同审批权限、印章使用审批权限、表决比例、章程版本日期、生效日期、是否有后续修正案。

每份章程或合伙协议必须生成权限摘要：

```text
章程合伙协议/<文件编号>/authority-summary.md
```

## 10. 股东出资管理

股东出资管理用于帮助企业记录股东认缴出资、实缴出资、出资期限、出资方式和逾期风险。

数据来源包括公司章程、章程修正案、股东协议、股东会决议、工商登记信息、出资证明书、验资报告、银行回单、财务凭证、股权转让协议。

`capital-contributions.csv` 字段包括：

```text
contribution_id,shareholder_name,shareholder_type,subscribed_amount,paid_amount,unpaid_amount,contribution_method,subscription_deadline,paid_date,equity_ratio,source_document,source_file_path,proof_file_path,status,risk_flags,next_action,created_at,updated_at
```

出资状态只能使用：

```text
未到期
临近到期
已实缴
部分实缴
逾期未缴
待复核
```

字段标准：

1. `shareholder_name` 必填。
2. `subscribed_amount` 应从章程、股东协议或工商登记信息中提取。
3. `paid_amount` 应从出资证明、验资报告、银行回单或财务凭证中提取。
4. `subscription_deadline` 能识别时必须写入提醒台账。
5. 未提供实缴证明时，不得断言已经实缴。
6. 章程、工商登记、股东协议数据不一致时，必须标记 `待复核`。
7. 出资期限已过但未见实缴证明时，必须标记高风险。

出资提醒包括认缴出资即将到期、认缴出资已逾期、部分实缴待补足、股东出资证明待补充、章程出资信息与工商登记不一致待复核。

当同时存在章程、股东协议、工商登记、出资证明等文件时，系统必须比对股东名称、认缴金额、出资比例、出资期限、出资方式、实缴证明、是否部分实缴或逾期未缴。比对结果保存为：

```text
<company_workspace>/03-公章证照/章程合伙协议/出资信息比对记录.md
```

## 11. 章程变更管理

上传章程修正案、新版章程、新合伙协议、合伙协议补充协议、改变权限结构的股东会或董事会决议时，必须触发章程变更记录。

每次变更必须记录变更日期、变更文件名称、变更前版本、变更后版本、变更事项摘要、涉及权限变化、是否影响担保/借款/重大合同/用印审批、需要复核的历史授权或审批事项、原始文件路径。

章程或合伙协议发生变更后，系统必须更新治理文件台账，生成新版权限摘要，保留旧版权限摘要，写入章程变更记录，检查是否影响现有授权和尚未完成的高风险用印事项，生成章程变更复核提醒。

## 12. 权限校验

对外承诺、担保、借款、融资、股权转让、重大资产处置、长期或大额合同、劳动解除或群体性员工事项、授权他人代表公司签署文件、用户明确要求“这个能不能盖章”时必须触发权限校验。

系统必须读取拟签署或拟盖章文件，识别文件类型和义务内容，提取金额、期限、相对方、承诺事项，读取当前有效章程、合伙协议或相关治理文件，检查对应事项审批权限，检查是否已有决议或授权文件，形成权限校验记录，给出内部建议动作。

权限校验结论只能使用：

```text
已通过
待补充审批
章程无明确依据
建议暂停用印
待人工确认
```

`authority-checks.csv` 字段至少包括：

```text
authority_check_id,matter_name,matter_type,related_file_path,amount,counterparty,governance_basis,required_approval,existing_approval,conclusion,risk_flags,record_md_path,created_at,updated_at
```

## 13. 公章证照提醒

本模块应生成证照到期、证照年检、授权文件失效、章程变更复核、印章停用或换刻后复核、高风险用印后续跟进、股东出资到期或逾期提醒。

所有提醒写入：

```text
<company_workspace>/04-提醒中心/reminders.csv
```

如飞书提醒已配置，由提醒模块同步日历。

## 14. 公章证照自然语言问答

用户可以询问公司有哪些证照快到期、哪些证照今年需要年检、某个授权文件是否有效、某枚公章由谁保管、某份担保合同能不能盖章、章程里对外担保谁审批、借款合同需要董事会还是股东会、最近一次章程变更改了什么、某个用印事项有没有审批记录、哪些股东出资快到期、某股东有没有实缴。

管理型问题先查 CSV 台账；权限判断型问题必须读取章程、合伙协议、授权文件和拟处理文件；回答必须标注来源；查不到不得编造；没有实缴证明时不得说“已实缴”。

## 15. 任务完成提示

每次任务结束必须输出已完成事项、证照/印章/授权台账位置、治理文件记录位置、权限校验记录、本次新增提醒、发现的风险或缺口、下一步建议。

## 16. 验收标准

1. 能导入营业执照、许可证、资质证书。
2. 能生成 `licenses.csv`。
3. 能导入印章清单并生成 `seals.csv`。
4. 能记录用印事项并生成 `seal-use.csv`。
5. 能导入授权文件并生成 `authorizations.csv`。
6. 能导入章程、合伙协议、决议并生成 `governance-documents.csv`。
7. 能提取章程或合伙协议中的权限规则。
8. 能生成章程权限摘要。
9. 能记录章程变更。
10. 章程变更后能提示复核现有授权和高风险用印事项。
11. 能从章程、股东协议、工商登记文件中提取股东出资信息。
12. 能生成 `capital-contributions.csv`。
13. 能记录认缴金额、实缴金额、出资期限和出资方式。
14. 能生成股东出资到期和逾期提醒。
15. 能比对章程、股东协议、工商登记、实缴证明之间的出资信息。
16. 能识别出资逾期、部分实缴、信息不一致等风险。
17. 没有实缴证明时不得断言股东已实缴。
18. 能对担保、借款、融资、重大合同、对外承诺进行权限校验。
19. 能输出明确权限校验结论。
20. 能生成证照到期、年检、授权失效、章程变更复核等提醒。
21. 能回答公章证照、章程权限和股东出资相关自然语言问题。
22. 不生成律师顾问版或正式法律意见。
23. 每次处理完成后告诉用户台账位置、记录位置、提醒状态和下一步。
